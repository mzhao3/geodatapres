What is a geodatabase?

A geodatabase is a relational database that stores geographic data. It is a container for storing spatial and attribute data and the relationships that exist among them. Geodatabases store information in vector data format. They come in many sizes and have varying numbers of users, and can be created, edited, and managed using ArcCatalog™ and ArcMap™.

These layers are examples of geographic data that might be found in a geodatabase. The geodatabase contains three primary dataset types: feature classes, raster datasets, and tables.

Feature class
- a table with a shape field containing point, line, or polygon geometries for geographic features.
Raster dataset
- contains rasters which represent continuous geographic phenomena
Table
- collection of rows, each containing the same fields
  • Feature classes are tables with shape fields.


Geodatabases are object relational. The information is stored as rows that have identities- these are the geodatabase objects. When the tables are stored in DBMS(database management systems) like Oracle, IBM DB2, PostgreSQL, or Microsoft SQL server, they are generally stored using a SQL spatial type, and can be accessed with SQL.

Types of Geodatabases

Geodatabases can scale from small, single-user databases built on files up to larger workgroup geodatabases accessible by many users. There are a few main types:

  • Personal geodatabases
  • File geodatabases
  • Enterprise geodatabases

Personal Geodatabases



  • Have a .mdb file extension (Microsoft Access database format).
    • Windows only
  • Has a 2GB size limitation.
  • Can be read by multiple people at a time, but edited by only one person at a time.
  • Better choice for outputs and analysis.
  • Data can be transferred to an enterprise geodatabase using ArcCatalog™.

File Geodatabases



  • Have a .gdb file extension.
  • Each dataset is a separate file on disk. All the datasets that belong to one geodatabase are contained in a single folder.
  • By default, each dataset can grow to 1TB.
  • Can be read by multiple people at a time, but edited by only one person at a time.
  • Better choice for editing and storing large amounts of information.

Enterprise Geodatabases

  • Suitable for large workgroups.
  • Can be read and edited by multiple users at the same time.
  • Can store both vector and raster data
  • Require ArcSDE software (allows you to view and work with data) and DBMS (stores spatial data).

Usage with Python

All geoprocessing tools can be accessed from Python using ArcPy. Simply include the following code in order to use it:
import arcpy
arcpy.env.workspace = "c:/PATH/file_name.gdb"
result = arcpy.function_name(parameter, parameter)

Advantages of Geodatabases

  • Multiple types of GIS data can be stored in a single geodatabase, making it easier to manage and access.
  • Supports 2-, 3-, and 4-D vector features.
  • Can be compressed to .zip files, so they are much better for datasets massive in size.
  • There is no limit to the geodatabase size (tables and feature classes have a limit of 1TB).

Drawbacks of Geodatabases

  • Geodatabases (especially file geodatabases) need an Esri environment to run. Once outside, it becomes less consumable.
    • Compare this to shapefiles, which can be consumed by almost 100% of GIS software.
  • Shapefiles are better for cross-platform sharing.
