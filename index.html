What is a geodatabase?

A geodatabase is a relational database that stores geographic data. It is a container for storing spatial and attribute data and the relationships that exist among them. Geodatabases store information in vector data format. They come in many sizes, can have varying numbers of users, and can be created, edited, and can be managed using programs like ArcCatalog™ and ArcMap™.

The layers in the image above are examples of the types of geographic data that might be found in a geodatabase. The geodatabase contains three primary dataset types: feature classes, raster datasets, and tables.

Feature Class
- a table with a shape field containing point, line, or polygon geometries for geographic features.
Raster Dataset
- contains rasters (in other words, the image is made up of pixels) which represent continuous geographic phenomena.
Table
- collection of rows, each containing the same fields.
  • Feature classes are tables with shape fields.


In this image, you can see the different types of dataset types that can be found in the geodatabase. In the upper left is a table, with the typical rows and columns. These tables contain information that can be queried. In the upper right is topology, represented by pixels in varying shades. This is a raster dataset. On the bottom are visual representations of feature classes, which is how they will appear on the map creatd by the geodatabase. In the database, they are stored as tables with an extra column that defines them as shapes, as can be seen in the image below.


Geodatabases are object relational. The information is stored as rows that have identities- these are the geodatabase objects. When the tables are stored in DataBase Management Systems (DMBM) like Oracle, IBM DB2, PostgreSQL, or Microsoft SQL server, they are generally stored using a SQL spatial type, and can be accessed with SQL.

Types of Geodatabases

Geodatabases can scale from small, single-user databases built on files up to larger workgroup geodatabases accessible by many users. There are a few main types:

  • Personal geodatabases
  • File geodatabases
  • Enterprise geodatabases

Personal Geodatabases



  • Have a .mdb file extension (Microsoft Access database format).
    • Windows only
  • Has a 2GB size limitation.
  • Can be read by multiple people at a time, but edited by only one person at a time.
  • Better choice for outputs and analysis.
  • Data can be transferred to an enterprise geodatabase using ArcCatalog™.

File Geodatabases



  • Have a .gdb file extension.
  • Each dataset is a separate file on disk. All the datasets that belong to one geodatabase are contained in a single folder.
  • By default, each dataset can grow to 1TB.
  • Can be read by multiple people at a time, but edited by only one person at a time.
  • Better choice for editing and storing large amounts of information.

Enterprise Geodatabases

  • Suitable for large workgroups.
  • Can be read and edited by multiple users at the same time.
  • Can store both vector and raster data
  • Require ArcSDE software (allows you to view and work with data) and DBMS (stores spatial data).

Usage with Python

All geoprocessing tools can be accessed from Python using ArcPy. Simply include the following code in order to use it:
import arcpy
arcpy.env.workspace = "c:/PATH/file_name.gdb"
result = arcpy.function_name(parameter, parameter)

Advantages of Geodatabases

  • Multiple types of GIS data can be stored in a single geodatabase, making it easier to manage and access.
  • Supports 2-, 3-, and 4-D vector features.
  • Can be compressed to .zip files, so they are much better for datasets massive in size.
  • There is no limit to the geodatabase size (tables and feature classes have a limit of 1TB).
  • Geographic features and attribute data can be separated.
  • Larger datasets are easier to access and manage.


The alternative to geodatabases are shapefiles, a digital vector storage format for storing geometric location and associated attribute information. This format lacks the capacity to store topological information (raster files). Shapefiles require three types of files:
  • .shp – this file stores the geometry of the feature
  • .shx – this file stores the index of the geometry
  • .dbf – this file stores the attribute information for the feature
All of the information is scattered across multiple files, which must be stored in the same location with the same name or else the shapefile will not load.

Drawbacks of Geodatabases

  • Geodatabases (especially file geodatabases) need specific programs like ArcCatalog in order to run. Once outside of this environment, it becomes less consumable.
    • Compare this to shapefiles, which can be consumed by almost 100% of GIS software.
  • Shapefiles are better for cross-platform sharing.
  • Databases require extra knowledge of administration, querying, managing joins/relates, etc.
